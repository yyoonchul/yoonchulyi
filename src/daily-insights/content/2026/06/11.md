## 📝 Detailed Notes

### 1. We Should Take Text Optimization More Seriously

1. Lee wants the research community to treat the mutable text layer as a real site of learning.
   - The post defines text optimization broadly: prompts, context, filesystem state, memory, retrieval databases, and model harnesses.
   - Lee argues that deployed AI systems are stateful compound systems, so learning can mean changing any behavior-conditioning state, not only model weights.
   - The core question is which update target fits a given piece of information, because weights and text artifacts have different costs, capacities, and failure modes.
2. The argument for text optimization starts with sample efficiency and inductive bias.
   - Lee frames good text edits as compact patches to a pretrained world prior, rather than as attempts to relearn structure from scratch.
   - Short, high-likelihood text specifications can encode broad regularities more cheaply than large collections of examples.
   - The post points to a recurring pattern where teams first elicit or compose behavior through the text layer, then later distill useful behavior into weights.
3. "Update-time compute" is the post's main new scaling axis.
   - Lee compares it to inference-time scaling: instead of spending more compute to answer one input, a system spends more compute to learn from one experience.
   - A failed trajectory can be reread, diagnosed, abstracted, compared against candidate revisions, and converted into a proposed text update.
   - This is especially relevant when failures are expensive, behavior is hard to specify, or offline traces are plentiful but awkward for standard supervised fine-tuning or offline RL.
4. Lee accepts several strong arguments for weights but reframes them as routing problems.
   - Weights are better for stable, repeatedly useful information because they amortize behavior without consuming context window space.
   - Text is better for information that is volatile, local, auditable, personalized, or not yet trusted enough to train into the model.
   - Good text-layer systems also avoid dumping everything into context; retrieval, memory, and progressive disclosure can condition behavior on a much larger state than the raw input window.
5. The post ends as a research agenda for the text layer.
   - Lee calls for theory that explains why text space can provide a better prior than weight space for some update problems.
   - Better evaluations should isolate text-layer benefits while detecting benchmark leakage, overfitting, and harness-enabled cheating.
   - Lee also wants architecture research, HCI work for "verbal fine-tuning," and larger-scale compute budgets for text optimization experiments.

## 📝 상세 정리

### 1. We Should Take Text Optimization More Seriously

1. Lee는 연구 community가 mutable text layer를 실제 learning site로 다뤄야 한다고 본다.
   - 글은 text optimization을 prompt, context, filesystem state, memory, retrieval database, model harness까지 넓게 정의한다.
   - Lee는 deployed AI system이 stateful compound system이므로 learning은 model weight뿐 아니라 behavior-conditioning state를 바꾸는 일도 포함한다고 주장한다.
   - 핵심 질문은 어떤 정보가 어떤 update target에 맞는가다. weight와 text artifact는 cost, capacity, failure mode가 서로 다르기 때문이다.
2. Text optimization의 첫 번째 근거는 sample efficiency와 inductive bias다.
   - Lee는 좋은 text edit을 structure를 처음부터 다시 학습하는 것이 아니라 pretrained world prior에 붙이는 compact patch로 본다.
   - 짧고 likelihood가 높은 text specification은 많은 example collection보다 넓은 regularity를 더 싸게 encode할 수 있다.
   - 글은 team들이 먼저 text layer로 behavior를 elicit하거나 compose한 뒤, 유용한 behavior를 나중에 weight로 distill하는 반복 패턴을 지적한다.
3. "Update-time compute"는 이 글의 핵심 scaling axis다.
   - Lee는 이를 inference-time scaling과 비교한다. 하나의 input에 답하기 위해 더 많은 compute를 쓰는 대신, 하나의 experience에서 배우기 위해 더 많은 compute를 쓰는 방식이다.
   - 실패한 trajectory는 다시 읽고, 진단하고, 추상화하고, candidate revision과 비교한 뒤 proposed text update로 바뀔 수 있다.
   - 이는 failure가 비싸거나, 원하는 behavior를 명시하기 어렵거나, offline trace는 많지만 standard supervised fine-tuning이나 offline RL에 쓰기 어색할 때 특히 중요하다.
4. Lee는 weight에 대한 강한 논거를 인정하지만 이를 routing problem으로 다시 해석한다.
   - Weight는 안정적이고 반복적으로 유용한 정보에 더 적합하다. context window를 쓰지 않고 behavior를 amortize할 수 있기 때문이다.
   - Text는 volatile, local, auditable, personalized한 정보나 아직 model에 training할 만큼 신뢰되지 않은 정보에 더 적합하다.
   - 좋은 text-layer system은 모든 정보를 context에 밀어 넣지도 않는다. retrieval, memory, progressive disclosure는 raw input window보다 훨씬 큰 state로 behavior를 condition할 수 있다.
5. 글은 text layer를 위한 research agenda로 끝난다.
   - Lee는 일부 update problem에서 text space가 weight space보다 더 나은 prior를 제공하는 이유를 설명하는 theory가 필요하다고 말한다.
   - 더 나은 evaluation은 text-layer benefit을 분리하면서 benchmark leakage, overfitting, harness-enabled cheating을 감지해야 한다.
   - Lee는 architecture research, "verbal fine-tuning"을 위한 HCI work, text optimization experiment를 위한 더 큰 compute budget도 요구한다.
