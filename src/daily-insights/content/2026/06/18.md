## 📝 Detailed Notes

### 1. Running local models is good now

1. Boykis says local models have crossed a practical usefulness threshold.
   - The author has used local models since early releases and says they are finally "surprisingly good" for everyday work.
   - The tested models include Mistral 7B, Gemma 3, GPT-OSS-20B, Qwen 3 MOE, Qwen 2.5 Coder, and other Qwen variants.
   - The tested setups include raw llama.cpp with Open WebUI, llama-cpp-python, Ollama, llamafiles, and LM Studio.
2. The turning point is measured by reduced dependence on API models.
   - Boykis uses a personal benchmark: whether local output still needs to be checked against a frontier API model.
   - GPT-OSS is described as the first local model that made that verification step much less frequent.
   - The newer Gemma 4 family made local agentic coding loops feel usable rather than merely interesting.
3. The reported hardware and task examples are concrete.
   - The machine is a 2022 M2 Mac with 64 GB RAM and 1 TB of storage.
   - Boykis says `gemma-4-26b-a4b` through LM Studio has become the default local model for the current setup.
   - The local agent refactored a notebook-style Python script into a 5-6 module repository, improved type hints, proofread posts, wrote unit tests, and bootstrapped a two-tower recommendation model repo.
4. The working setup pairs local inference with a constrained agent harness.
   - Boykis currently uses Pi as the agent harness and LM Studio as the inference server.
   - The harness talks to LM Studio through an OpenAI-compatible local endpoint at `http://host.docker.internal:1234/v1`.
   - Agentic sessions run inside Docker with restricted execution so the agent can work in a mounted workspace without direct host-level access.
5. The conclusion is optimistic but not production-maximalist.
   - Remaining problems include slower inference, hardware-limited context windows, RAM pressure, and occasional prompt-template mismatches.
   - The ecosystem has become easier through LM Studio, Hugging Face distribution, model buttons, quantized releases, and fast patching of early model issues.
   - The biggest advantage is inspectability: users can watch token flow, change context windows, alter prompts and quantization, and compare local models directly.

### 2. How to Pivot Well

1. The article argues that AI has made pivot judgment more important.
   - The post opens with familiar stories such as Slack, Instagram, and Twitter to show why founders romanticize pivots.
   - It asks whether those web-era examples still apply when AI model capabilities change every quarter.
   - The central claim is that founders now need sharper judgment about when to pivot, how quickly to move, and what to move toward.
2. The definition of a good pivot keeps one thing planted.
   - The basketball analogy is that a pivot keeps one foot in place; moving both feet is traveling.
   - In a startup, the planted foot might be the team, investor capital, customer learning, or a specific insight from the previous attempt.
   - Emily Bennett warns that the worst failure can be continuing with determination after it is already clear the company is aimed at the wrong thing.
3. The case studies show three distinct pivot patterns.
   - Clay began with a broad programming-access mission, then noticed that growth teams were the most active users and rebuilt around go-to-market workflows.
   - Lovable emerged after an open-source command-line code-generation project gained about 50,000 GitHub stars in roughly two months, but the real market was non-technical builders.
   - Cursor left AI autocomplete for CAD after recognizing weak founder-market fit and moved to coding, where the team had more passion, more data, and clearer customer empathy.
4. The timing advice focuses on market pull rather than founder attachment.
   - Bennett says founders can become so excited by a concept that they miss weak commercial appetite.
   - Troy Kirwin suggests trying to schedule fifty customer conversations in two weeks and treating low response as a signal.
   - A rough 15-20% conversion rate for those conversations is presented as a warning threshold for whether the market wants to engage.
5. The execution advice turns prior learning into the next test.
   - Bennett says previous customer conversations are not wasted because positive insights can become the foundation for a new direction.
   - The suggested process starts with ten deep customer conversations, then expands into A/B tests on messaging and customer archetypes.
   - Modern prototype tools reduce the excuse to delay feedback, and investors may see a thoughtful pivot as evidence that the team is moving fast and seeking truth.

### 4. Social media's next evolution: user-controlled algorithms

1. The article describes a shift in recommendation control.
   - Social platforms historically let users follow, like, or hide content while the platform's ranking model decided the feed.
   - TechCrunch says newer tools let users directly personalize recommendation systems with AI assistance.
   - The analogy is a move away from a one-size-fits-all TV channel toward a streaming-style feed that users can tune.
2. Threads is moving from public algorithm requests to private controls.
   - Threads launched "Your Algo" on June 16, 2026, building on the earlier "Dear Algo" tool from February.
   - "Dear Algo" required users to publish public posts that told the algorithm what to show more or less often.
   - "Your Algo" lets users make those preferences privately and set how long the change should last: one, three, or seven days.
3. Instagram is exposing recommendation topics across more surfaces.
   - Instagram's "Your Algorithm" tool lets users see topics that shape recommendations and adjust those interests.
   - The tool first launched for Reels in December 2025 and is now available across feed, Explore, and Reels.
   - Adam Mosseri has argued that large language models can make ranking systems more understandable and more directly steerable.
4. TikTok offers topic sliders and AI-assisted keyword filtering.
   - TikTok's "Manage Topics" tool, launched in 2024, lets users tune categories such as sports, travel, humor, current affairs, dance, and food.
   - The interface uses sliders to adjust how much of each topic should appear in the For You feed.
   - In 2025, TikTok added AI-powered Smart Keyword Filters that also limit related terms and synonyms.
5. The incentives align user control with platform engagement.
   - Users get a feed that better matches current interests, moods, and tolerance for specific topics.
   - Platforms can increase engagement by reducing irrelevant or unwanted content without giving up recommendation systems entirely.
   - The likely competitive frontier is how much transparency and control platforms provide without making feeds feel complicated.

### 6. Agentic coding and persistent returns to expertise

1. Anthropic introduces a large-scale measurement framework for Claude Code usage.
   - The report analyzes about 400,000 interactive sessions from roughly 235,000 people between October 2025 and April 2026.
   - It focuses on Claude Code usage through a command-line interface, Claude.ai, and the Claude Code desktop app.
   - The analysis is privacy-preserving and classifies work modes, decision ownership, apparent expertise, occupations, and success signals.
2. The core division of labor is planning by humans and execution by Claude.
   - Users make about 70% of planning decisions, such as what to do, which approach to take, and what counts as done.
   - Claude makes about 80% of execution decisions, such as which files to change, what code to write, and which commands to run.
   - A typical session has about four turns, and each user prompt triggers about 10 Claude actions on average.
3. More expert users elicit more work from the agent.
   - Anthropic rates task-specific expertise from novice to expert using signals such as precision of direction and correction behavior.
   - Novice sessions trigger about five Claude actions and 600 output words per prompt.
   - Expert sessions trigger about 12 actions and 3,200 output words per prompt, with statistically significant trends after controls.
4. Claude Code work is broadening beyond fixing broken code.
   - About 56% of sessions involve writing, fixing, testing, or orchestrating code.
   - Operating software accounts for 17%, planning or exploration accounts for 14%, and analysis or prose accounts for 13%.
   - From October 2025 to April 2026, fixing broken code fell from 33% to 19%, while operating software, data analysis, and writing grew.
5. Success depends more on domain expertise than formal coding occupation.
   - Novice-rated sessions reach verified success 15% of the time and at least partial success 77% of the time.
   - Intermediate-or-higher sessions reach verified success 28-33% of the time and partial success 91-92% of the time.
   - In code-producing sessions, non-software occupations reach partial success at 88%, close to 89% for software-related occupations.

### 4. Social media's next evolution: user-controlled algorithms

1. 글은 recommendation control의 이동을 설명한다.
   - Social platform은 전통적으로 사용자가 follow, like, hide를 하게 했지만 feed 결정은 platform ranking model이 맡았다.
   - TechCrunch는 새로운 tool이 사용자가 AI의 도움으로 recommendation system을 직접 personalize하게 한다고 말한다.
   - 비유하자면 one-size-fits-all TV channel에서 사용자가 조정할 수 있는 streaming-style feed로 이동하는 흐름이다.
2. Threads는 공개 algorithm 요청에서 private control로 이동하고 있다.
   - Threads는 2026년 6월 16일 "Your Algo"를 출시했으며, 이는 2월에 나온 "Dear Algo" tool을 확장한 것이다.
   - "Dear Algo"는 사용자가 algorithm에 더 많이 또는 덜 보고 싶은 내용을 공개 post로 말해야 했다.
   - "Your Algo"는 그런 preference를 비공개로 설정하고 변경 기간을 1일, 3일, 7일 중 선택하게 한다.
3. Instagram은 더 많은 surface에서 recommendation topic을 드러낸다.
   - Instagram의 "Your Algorithm" tool은 사용자가 recommendation을 형성하는 topic을 보고 관심사를 조정하게 한다.
   - 이 tool은 2025년 12월 Reels용으로 먼저 출시됐고, 이제 feed, Explore, Reels 전반에서 사용할 수 있다.
   - Adam Mosseri는 large language model이 ranking system을 더 이해하기 쉽게 만들고 사용자가 preference를 명시적으로 전달하게 할 수 있다고 주장했다.
4. TikTok은 topic slider와 AI-assisted keyword filtering을 제공한다.
   - 2024년에 출시된 TikTok의 "Manage Topics" tool은 sports, travel, humor, current affairs, dance, food 같은 category를 조정하게 한다.
   - interface는 slider를 사용해 For You feed에 각 topic이 얼마나 많이 나올지 조정한다.
   - 2025년 TikTok은 관련 단어와 synonym까지 제한하는 AI-powered Smart Keyword Filters를 추가했다.
5. incentive는 user control과 platform engagement를 함께 정렬한다.
   - 사용자는 현재 관심사, 기분, 특정 topic에 대한 tolerance에 더 잘 맞는 feed를 얻는다.
   - platform은 recommendation system을 완전히 포기하지 않으면서 원치 않거나 관련 없는 content를 줄여 engagement를 높일 수 있다.
   - 경쟁의 다음 지점은 feed를 복잡하게 만들지 않으면서 얼마나 많은 transparency와 control을 제공하느냐가 될 가능성이 크다.

### 6. Agentic coding and persistent returns to expertise

1. Anthropic은 Claude Code 사용에 대한 대규모 measurement framework를 제시한다.
   - 보고서는 2025년 10월부터 2026년 4월까지 약 235,000명이 진행한 interactive session 약 400,000건을 분석한다.
   - 분석 대상은 command-line interface, Claude.ai, Claude Code desktop app을 통한 Claude Code 사용이다.
   - 분석은 privacy-preserving 방식이며 work mode, decision ownership, apparent expertise, occupation, success signal을 분류한다.
2. 핵심 division of labor는 사람이 planning을, Claude가 execution을 맡는 구조다.
   - 사용자는 무엇을 할지, 어떤 approach를 택할지, 무엇을 완료로 볼지 같은 planning decision의 약 70%를 내린다.
   - Claude는 어떤 file을 바꿀지, 어떤 code를 쓸지, 어떤 command를 실행할지 같은 execution decision의 약 80%를 내린다.
   - 일반적인 session은 약 4개의 turn으로 구성되며, user prompt 하나는 평균 약 10개의 Claude action을 유발한다.
3. expertise가 높은 사용자는 agent로부터 더 많은 작업을 끌어낸다.
   - Anthropic은 direction의 정밀도와 correction behavior 같은 signal을 사용해 task-specific expertise를 novice부터 expert까지 평가한다.
   - novice session은 prompt 하나당 Claude action 약 5개와 output word 약 600개를 유발한다.
   - expert session은 prompt 하나당 action 약 12개와 output word 약 3,200개를 유발하며, 통제 후에도 trend는 statistically significant하다.
4. Claude Code 작업은 broken code fixing을 넘어 넓어지고 있다.
   - session의 약 56%는 code 작성, 수정, testing, orchestration과 관련된다.
   - operating software는 17%, planning 또는 exploration은 14%, analysis 또는 prose는 13%를 차지한다.
   - 2025년 10월부터 2026년 4월까지 broken code fixing 비중은 33%에서 19%로 줄었고, operating software, data analysis, writing은 늘었다.
5. success는 formal coding occupation보다 domain expertise에 더 의존한다.
   - novice-rated session은 verified success 15%, at least partial success 77%에 도달한다.
   - intermediate 이상 session은 verified success 28-33%, partial success 91-92%에 도달한다.
   - code-producing session에서 non-software occupation은 partial success 88%에 도달해 software-related occupation의 89%와 가깝다.
