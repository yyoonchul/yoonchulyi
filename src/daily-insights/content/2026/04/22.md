### 4. Chronicle – Codex

1. Chronicle is positioned as a memory-extension layer rather than a generic screen recorder.
   - OpenAI describes it as a way to build Codex memories from recent screen context so users do not have to restate as much background in every prompt.
   - The page says Chronicle can help Codex understand what is on screen, identify the right source to use, and remember tools and workflows over time.
   - It is therefore framed as context compression for ongoing work, not as a standalone surveillance or logging feature.
2. Availability is narrow and the setup is explicit.
   - Chronicle is currently described as an opt-in research preview for ChatGPT Pro subscribers on macOS.
   - The page also says it is not available in the EU, UK, or Switzerland.
   - Enabling it requires turning on Memories in Codex settings and granting both Screen Recording and Accessibility permissions in macOS.
3. OpenAI gives users ongoing control over when Chronicle is active.
   - The docs say users can pause or resume Chronicle from the Codex menu bar icon.
   - They also recommend pausing it before meetings or while viewing sensitive content that should not become memory context.
   - Full disablement lives under `Settings > Personalization > Memories`, where Chronicle can be turned off entirely.
4. The documentation is unusually concrete about local storage and retention.
   - Temporary screen captures may appear under `$TMPDIR/chronicle/screen_recording/` while Chronicle runs.
   - The page says captures older than six hours are deleted while Chronicle is running.
   - Generated Chronicle memories are stored locally as unencrypted markdown under `$CODEX_HOME/memories_extensions/chronicle/`, and users can inspect, edit, or delete them.
5. The main tradeoffs are cost, privacy, and attack surface.
   - OpenAI says Chronicle uses sandboxed background agents to generate memories and that these agents consume rate limits quickly.
   - The docs warn that screen-based context increases prompt-injection risk because malicious instructions visible on screen may influence Codex.
   - OpenAI also states that screenshots are processed on its servers to generate memories but are not stored on those servers after processing unless required by law, and are not used for training.

### 4. Chronicle – Codex

1. Chronicle은 일반적인 화면 녹화기가 아니라 memory extension 계층으로 위치 지어집니다.
   - OpenAI는 이를 최근 화면 맥락에서 Codex 메모리를 만들어, 사용자가 매 프롬프트마다 배경을 길게 다시 쓰지 않게 해 주는 기능으로 설명합니다.
   - 문서에 따르면 Chronicle은 화면에 보이는 내용을 이해하고, 적절한 원문 소스를 찾고, 시간이 지나도 도구와 워크플로를 기억하는 데 도움을 줍니다.
   - 따라서 이것은 독립형 감시 기능이 아니라 ongoing work를 위한 context compression 계층으로 제시됩니다.
2. 제공 범위는 좁고 설정 절차는 명시적입니다.
   - Chronicle은 현재 ChatGPT Pro 구독자의 macOS용 Codex에서 제공되는 opt-in research preview로 설명됩니다.
   - 또한 EU, UK, Switzerland에서는 아직 사용할 수 없다고 적혀 있습니다.
   - 활성화하려면 Codex 설정에서 Memories를 켜고 macOS의 Screen Recording 및 Accessibility 권한을 모두 허용해야 합니다.
3. OpenAI는 사용자가 Chronicle의 활성 여부를 계속 제어할 수 있다고 설명합니다.
   - 문서에 따르면 사용자는 Codex 메뉴 막대 아이콘에서 Chronicle을 일시정지하거나 다시 시작할 수 있습니다.
   - 또 회의 중이거나 민감한 내용을 볼 때는 해당 내용이 메모리 맥락이 되지 않도록 Chronicle을 일시정지하라고 권합니다.
   - 완전 비활성화는 `Settings > Personalization > Memories`에서 Chronicle을 꺼서 처리할 수 있습니다.
4. 문서는 로컬 저장과 보존 정책을 매우 구체적으로 적고 있습니다.
   - 임시 화면 캡처는 Chronicle 실행 중 `$TMPDIR/chronicle/screen_recording/` 아래에 나타날 수 있습니다.
   - 문서에 따르면 6시간보다 오래된 캡처는 Chronicle이 실행 중일 때 삭제됩니다.
   - 생성된 Chronicle 메모리는 `$CODEX_HOME/memories_extensions/chronicle/` 아래 로컬의 암호화되지 않은 markdown으로 저장되며, 사용자가 직접 확인, 수정, 삭제할 수 있습니다.
5. 핵심 트레이드오프는 비용, 프라이버시, 공격 표면입니다.
   - OpenAI는 Chronicle이 sandboxed background agent를 써서 메모리를 생성하며, 이 에이전트들이 rate limit을 빠르게 소비한다고 설명합니다.
   - 또한 화면 기반 맥락은 악성 지시문이 화면에 보일 경우 Codex에 영향을 줄 수 있으므로 prompt injection 위험을 높인다고 경고합니다.
   - OpenAI는 스크린샷이 메모리 생성을 위해 서버에서 처리되지만, 법적 요구가 없는 한 처리 후 서버에 저장되지 않으며 학습에도 사용되지 않는다고 밝힙니다.
