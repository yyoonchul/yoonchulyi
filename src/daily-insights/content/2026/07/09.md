## 📋 Quick Summary

### I Think I Have LLM Burnout
**Source:** Alec Scollon Blog · **Author:** Alec Scollon · **Category:** AI · **Link:** [Original](https://www.alecscollon.com/blog/llm-burnout/)
- Alec Scollon describes a year of heavy LLM use across work and home, including Claude Code, Codex, ChatGPT, Gemini overviews, and unsupervised code-generation review.
- Scollon says the developer job now includes designing code, explaining designs to an LLM, reviewing generated code, and then writing or revising code.
- The main fatigue comes less from isolated mistakes than from repeated patterns: false assumptions, hallucinations, emphatic fragments, emoji-heavy style, and a sameness that compounds over time.

---

## 📝 Detailed Notes

### 1. I Think I Have LLM Burnout

1. Scollon frames the problem as ordinary but constant LLM exposure.
   - Scollon says daily LLM usage is probably average by current developer standards, not unusually advanced.
   - The workflow is still mostly task-by-task conversation with Claude Code at work and Codex at home.
   - Scollon sometimes lets the assistant write code, but still reads the output, understands it, and revises it.
   - The author is not describing deep autonomous-agent orchestration, but still reads AI-generated text for hours each day.
2. The development workflow has shifted from direct implementation to LLM-mediated work.
   - Scollon says the job has moved from designing and writing code to designing code, explaining that design to an LLM, reviewing generated code, and then writing code.
   - The LLM steps expose the author to approaches that might not have been considered otherwise.
   - They also make unfamiliar technical areas feel more approachable.
   - The productivity benefit is real enough that Scollon does not expect to stop using LLMs.
3. The author's current project intensifies the exposure.
   - Scollon is working on a framework for large-scale, unsupervised code generation in the codebase.
   - When not using Claude to build tooling, the author reviews output from an unsupervised agent based on Qwen.
   - That means both tooling creation and output review involve reading LLM-generated material.
   - The article treats this as an ambient work condition rather than a one-off experiment.
4. General information-seeking has also moved toward AI answers.
   - For casual questions, Scollon will often ask ChatGPT or read Gemini's overview unless there is already a known site to check.
   - The author still falls back to browsing when an LLM answer is wrong.
   - AI summaries are useful enough for many casual queries, especially when search results are cluttered with low-quality AI-generated articles.
   - This extends LLM exposure beyond programming into routine research and everyday lookup behavior.
5. The burnout comes from repetitive style and repetitive failure modes.
   - Scollon says the feeling changed over the last few months, creating a small dread before reading LLM output.
   - The expected problems include false assumptions, hallucinations, clipped emphatic phrasing, and excessive emojis.
   - The author is not condemning LLMs wholesale and notes that humans can also be unreliable or annoying.
   - The distinctive issue is repetition: LLMs often write in similar styles and make similar mistakes, and that repeated sameness has become exhausting.

## 📝 상세 정리

### 1. I Think I Have LLM Burnout

1. 스콜런은 문제를 특별한 고급 사용이 아니라 일상적이고 지속적인 대규모 언어 모델 노출로 설명한다.
   - 스콜런은 현재 개발자 기준으로 자신의 대규모 언어 모델 사용량이 아마 평균 수준이며, 사용 방식도 특별히 고도화되어 있지는 않다고 말한다.
   - 작업 방식은 여전히 업무에서는 클로드 코드(Claude Code), 집에서는 코덱스(Codex)와 한 번에 하나의 작업을 두고 대화하는 형태에 가깝다.
   - 스콜런은 때때로 보조 도구가 코드를 쓰게 하지만, 그 결과물을 꼼꼼히 읽고 이해한 뒤 수정한다.
   - 글은 깊은 자율 에이전트(autonomous agent) 오케스트레이션(orchestration)을 다루는 것이 아니지만, 그래도 스콜런은 매일 몇 시간씩 AI 생성 텍스트를 읽는다.
2. 개발 흐름은 직접 구현 중심에서 대규모 언어 모델을 거치는 방식으로 바뀌었다.
   - 스콜런은 일이 코드를 설계하고 작성하는 것에서, 코드를 설계하고 그 설계를 대규모 언어 모델에 설명하고 생성된 코드를 검토한 뒤 다시 코드를 쓰는 방식으로 옮겨 갔다고 말한다.
   - 대규모 언어 모델을 거치는 단계는 스콜런이 혼자서는 떠올리지 못했을 수도 있는 접근법을 보여준다.
   - 이 단계들은 깊은 지식이 없는 기술 영역도 더 다루기 쉽게 느끼게 한다.
   - 생산성 이점은 분명해서 스콜런은 대규모 언어 모델 사용을 멈출 생각은 없다고 본다.
3. 현재 프로젝트는 이러한 노출을 더 강하게 만든다.
   - 스콜런은 코드베이스(codebase) 안에서 대규모 비감독 코드 생성(large-scale unsupervised code generation)을 위한 프레임워크(framework)를 세우는 일을 하고 있다.
   - 클로드(Claude)로 도구를 만들지 않을 때는 큐원(Qwen)을 기반으로 한 비감독 에이전트(unsupervised agent)의 출력물을 검토한다.
   - 그래서 도구 제작과 출력 검토 모두 대규모 언어 모델이 만든 자료를 읽는 일과 연결된다.
   - 글은 이를 일회성 실험이 아니라 주변을 계속 채우는 업무 조건으로 다룬다.
4. 일반적인 정보 탐색도 AI 답변 쪽으로 옮겨 갔다.
   - 스콜런은 확인하고 싶은 사이트를 이미 알고 있지 않은 한, 가벼운 질문에는 챗GPT(ChatGPT)에 묻거나 제미나이(Gemini) 개요를 읽는 경우가 많다고 말한다.
   - 대규모 언어 모델의 답이 틀리면 여전히 브라우징(browsing)으로 돌아간다.
   - 특히 검색 결과가 품질 낮은 AI 생성 글로 어지러울 때, AI 요약은 많은 가벼운 질문에 충분히 쓸 만하다.
   - 이 변화는 대규모 언어 모델 노출을 프로그래밍 밖의 일상 조사와 정보 확인 행동으로까지 넓힌다.
5. 소진감은 반복되는 문체와 반복되는 실패 양식에서 나온다.
   - 스콜런은 지난 몇 달 사이 감정이 달라졌고, 대규모 언어 모델 출력을 읽기 전 작은 꺼림칙함이 생겼다고 말한다.
   - 예상되는 문제에는 잘못된 가정, 환각(hallucination), 끊어지는 듯한 강조 표현, 과도한 이모지(emoji)가 포함된다.
   - 스콜런은 대규모 언어 모델 전체를 비난하려는 것은 아니며, 사람도 믿기 어렵거나 성가실 수 있다고 덧붙인다.
   - 핵심 문제는 반복이다. 대규모 언어 모델은 비슷한 문체로 쓰고 비슷한 실수를 반복하며, 그 반복되는 비슷함이 스콜런을 지치게 만들었다.
