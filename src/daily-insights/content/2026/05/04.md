## 📝 Detailed Notes

### 1. From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company (OneManCompany)

1. The paper reframes multi-agent systems as an organization-design problem rather than an agent-capability problem.
   - Authors are from HUAWEI Noah's Ark Lab, University College London, and University of Liverpool.
   - The thesis: gains now come from how an agent workforce is assembled, coordinated, and improved over time, not from a smarter single agent.
   - OMC sits as an organizational layer above existing frameworks like LangGraph and Claude Code.
2. Talent-Container architecture decouples identity from runtime.
   - Talent encapsulates role, prompts, skills, and tools; Container is the execution backend (LangGraph, Claude Code, scripts).
   - Six typed organizational interfaces standardize how agents talk to platforms, so a Talent can be lifted across Containers without modification.
   - This is what enables a "digital talent market" where verified agent implementations can be sourced from the community.
3. Explore-Execute-Review (E²R) tree search governs project execution.
   - Models a project as dynamic search over organizational strategies with DAG-based task decomposition and AND-tree semantics.
   - A finite state machine governs task lifecycle, and the system provides formal guarantees on termination and deadlock freedom.
   - Seven invariants are enforced: DAG structure, mutual exclusion, schedule idempotency, review termination, cascade completeness, dependency completeness, and recovery correctness.
4. Self-evolution is wired in at two scopes.
   - Individual: CEO one-on-ones and post-task reflection update each agent's working principles.
   - Organizational: project retrospectives produce reusable Standard Operating Procedures.
   - A formal HR pipeline includes performance reviews, Performance Improvement Plans, and automated offboarding, mirroring real-company HR.
5. Benchmarks and case studies show cross-domain reach.
   - PRDBench: OMC hits 84.67% on project-level software development vs. 69.19% for Claude-4.5 (+15.48pp), at ~$6.91 per task ($345.59 over 50 tasks).
   - Case studies span content generation (GitHub trend report, $4.48), game dev with human-in-the-loop, illustrated audiobook production ($1.57 for 16 scenes), and an autonomous research survey ($16.26, 1 hour).
   - Human involvement is reduced to the initial prompt and optional candidate selection — planning, execution, and delivery run autonomously.

### 2. From Skill Text to Skill Structure (SSL Representation)

1. The paper diagnoses a representation gap in agent skill libraries.
   - Today's skills are stored as "text-heavy artifacts" — markdown docs, prompts, READMEs — that LLMs must re-read at runtime.
   - This makes skill discovery brittle and risk assessment opaque, especially as catalogs scale.
   - Authors are from Peking University.
2. SSL is a three-layer structured representation.
   - Scheduling Layer: invocation-level interfaces and dependencies (how the skill is called and what it needs).
   - Structural Layer: scene-based execution phases (the rough storyline of running the skill).
   - Logical Layer: atomic actions with resource boundaries (the smallest verifiable steps).
3. The framing borrows from classical linguistic and cognitive theories.
   - Memory Organization Packets, Script Theory, and Conceptual Dependency are cited as inspirations.
   - The normalizer is itself an LLM pipeline, but it is constrained to ground all extracted fields in source text rather than infer them.
   - This grounding constraint is meant to keep the structured form trustworthy as a substitute for the original docs.
4. Empirical gains are reported on two downstream tasks.
   - Skill Discovery: Mean Reciprocal Rank rises from 0.573 to 0.707 (+23% relative).
   - Risk Assessment: macro F1 rises from 0.744 to 0.787 (+5.8% relative).
   - For risk, combining source documents with SSL beat either alone, suggesting they carry complementary signal.
5. Released resources position SSL as a benchmarking baseline.
   - 6,184-skill corpus normalized into SSL form.
   - 403 task-grounded queries for retrieval evaluation.
   - 500 skills annotated with risk labels for safety evaluation.

### 3. The best AI dictation apps, tested and ranked

1. The category has shifted from raw STT to LLM-polished dictation.
   - LLM post-processing now removes filler words, fixes stumbles, and applies punctuation automatically.
   - The article frames this as a meaningful reduction in manual editing relative to older speech-to-text tools.
   - Author Ivan Mehta tests 11 apps across macOS, Windows, Linux, iOS, and web.
2. Subscription leaders cluster in the $8–$15/month range.
   - Wispr Flow: from $15/month, custom words and coding-style variable recognition, with a 2,000 words/week free desktop tier.
   - Willow: from $15/month, generates full passages from partial dictation and stores locally for privacy.
   - Aqua: from $8/month annually, claims fast latency and exposes its own speech-to-text API.
3. A distinct lifetime / one-time-purchase tier exists, mostly Mac-first.
   - Superwhisper: $249.99 lifetime option alongside $8.49/month and $84.99/year.
   - Dictato: €9.99 (~$12) lifetime with 2-year updates, Apple Intelligence integration, ~80ms latency.
   - VoiceInk: $25–$49 lifetime depending on device count, with global shortcuts and context-aware formatting.
4. Open-source and offline-first options anchor the privacy end.
   - Handy: free, open-source, push-to-talk, supports Mac/Windows/Linux.
   - VoiceTypr: 3-day trial then $35–$98 lifetime, offline-first with 99+ language support.
   - Monologue: downloadable model for offline use plus an optional Monokey hardware accessory.
5. A few apps target adjacent use cases beyond raw dictation.
   - AudioPen ($33 / 3 months up to $159 / 2 years): audio-note storage, style switching, note summarization, file uploads.
   - Typeless: 4,000 free words/week, claims no data retention, can rewrite full sentences.
   - Pricing tiers and word-count caps remain the main differentiators once transcription quality converges.

## 📝 상세 정리

### 1. From Skills to Talent: Organising Heterogeneous Agents as a Real-World Company (OneManCompany)

1. 논문은 멀티 에이전트 시스템을 에이전트 능력 문제가 아닌 조직 설계 문제로 다시 정의한다.
   - 저자는 화웨이 노아의 방주 연구소, UCL, 리버풀 대학 소속이다.
   - 핵심 주장은, 이제 성능 향상은 더 똑똑한 단일 에이전트가 아니라 에이전트 인력을 어떻게 구성·조정·개선하느냐에서 나온다는 것이다.
   - OMC는 LangGraph, Claude Code 같은 기존 프레임워크 위에 얹히는 조직 계층 역할을 한다.
2. Talent-Container 아키텍처가 정체성과 런타임을 분리한다.
   - Talent에는 역할, 프롬프트, 스킬, 도구가 묶이고 Container는 실행 백엔드(LangGraph, Claude Code, 스크립트)에 해당한다.
   - 여섯 개의 타입화된 조직 인터페이스로 에이전트-플랫폼 상호작용을 표준화해, 같은 Talent를 수정 없이 다른 Container로 옮길 수 있다.
   - 이 구조 위에서 검증된 에이전트 구현을 커뮤니티가 공급하는 "디지털 인재 시장"이 작동한다.
3. Explore-Execute-Review(E²R) 트리 탐색이 프로젝트 실행을 통제한다.
   - 프로젝트를 조직 전략에 대한 동적 탐색으로 모델링하고, DAG 기반 과제 분해와 AND-트리 의미론을 적용한다.
   - 유한 상태 기계가 과제 생애주기를 관리하며, 시스템은 종료성과 데드락 부재에 대한 형식적 보장을 제공한다.
   - 일곱 가지 불변식(DAG 구조, 상호 배제, 스케줄 멱등성, 리뷰 종료, 캐스케이드 완전성, 의존성 완전성, 복구 정합성)이 강제된다.
4. 자기 진화는 두 층위로 내장된다.
   - 개인 단위에서는 CEO와의 1:1과 과제 후 리플렉션이 각 에이전트의 작업 원칙을 갱신한다.
   - 조직 단위에서는 프로젝트 회고가 재사용 가능한 표준 운영 절차(SOP)로 쌓인다.
   - 성과 평가, PIP, 자동 오프보딩까지 포함한 정식 HR 파이프라인이 실제 회사 운영을 흉내 낸다.
5. 벤치마크와 사례가 도메인 횡단성을 보여 준다.
   - PRDBench에서 OMC는 프로젝트 단위 소프트웨어 개발 성공률 84.67%로 Claude-4.5(69.19%)를 +15.48%p 앞서며, 50개 과제 합산 $345.59(과제당 약 $6.91)로 처리한다.
   - 사례에는 콘텐츠 생성(깃허브 트렌드 리포트 $4.48), 인간 평가자가 개입한 게임 개발, 16개 장면 일러스트 오디오북 제작($1.57), 1시간 만에 끝낸 자율 리서치 서베이($16.26)가 포함된다.
   - 사람이 개입하는 지점은 초기 프롬프트 입력과 선택적 후보 검토에 한정되고, 기획·실행·납품은 자율 동작한다.

### 2. From Skill Text to Skill Structure (SSL Representation)

1. 논문은 에이전트 스킬 라이브러리에 표현 격차가 있다고 진단한다.
   - 현재 스킬은 마크다운 문서·프롬프트·README 같은 "텍스트 위주 산출물"로 저장되고, LLM이 런타임마다 다시 읽어야 한다.
   - 카탈로그 규모가 커질수록 스킬 검색은 취약해지고 리스크 평가는 불투명해진다.
   - 저자는 베이징대 소속이다.
2. SSL은 3계층 구조 표현이다.
   - Scheduling 계층: 호출 단위 인터페이스와 의존성을 다룬다(어떻게 호출되고 무엇이 필요한가).
   - Structural 계층: 장면 기반 실행 단계로, 스킬을 돌리는 큰 흐름에 해당한다.
   - Logical 계층: 자원 경계가 명시된 원자 단위 액션, 즉 검증 가능한 최소 단위다.
3. 프레이밍은 고전적 언어·인지 이론에서 빌려 왔다.
   - Memory Organization Packets, Script Theory, Conceptual Dependency가 영감 원천으로 인용된다.
   - 정규화기는 LLM 파이프라인이지만, 추출 필드를 추론이 아닌 원문 텍스트에 근거(grounding)하도록 제약된다.
   - 이 근거 제약이 SSL을 원문 문서의 신뢰 가능한 대체물로 만드는 핵심이다.
4. 두 다운스트림 과제에서 정량 향상이 보고된다.
   - 스킬 검색: MRR이 0.573에서 0.707로 약 23% 상대 향상한다.
   - 리스크 평가: macro F1이 0.744에서 0.787로 약 5.8% 상대 향상한다.
   - 리스크 평가에서는 원문 문서와 SSL을 함께 쓸 때 한쪽만 쓸 때보다 좋아, 둘이 보완적 신호를 담는다고 시사한다.
5. 공개 자원이 SSL을 벤치마킹 기준선으로 삼게 만든다.
   - 6,184건 스킬을 SSL 형식으로 정규화한 코퍼스.
   - 검색 평가용 403건 과제 기반 질의.
   - 안전성 평가용 리스크 라벨 500개가 부착된 스킬 세트.
