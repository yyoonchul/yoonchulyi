# Card News Skill

Convert a day's bilingual digest into an Instagram card news deck with contextual backgrounds. Runs after the `digest` skill has completed.

Each deck is **1 cover + one content slide per summary bullet**. So an article with 3 summary bullets fans out to 3 content slides (each with its own Korean header, all sharing the same background image). The cover title and per-bullet headers are short Korean lines pre-generated by this skill before the renderer runs.

## Inputs

- `content/YYYY/MM/DD.md`: existing bilingual digest (must exist).
- Optional arg: `datePath` in `YYYY/MM/DD` form. Defaults to today (KST).
- Optional flag: `--reuse-queries` to skip the query generation step when the queries sidecar already exists.

## Outputs

- `meta/card-news/headers/YYYY/MM/DD.json`: Korean short headers (per-bullet + one main cover title).
- `meta/card-news/queries/YYYY/MM/DD.json`: English image search queries.
- `meta/card-news/output/YYYY/MM/DD/NN.png`: slides in upload order.
- `meta/card-news/output/YYYY/MM/DD/credits.json`: per-slide image metadata.

## Execution Flow

### 1. Resolve target date

- Use the `datePath` argument if provided; otherwise compute today in KST.
- Verify `content/<datePath>.md` exists. If not, abort: `⚠️ No digest found for <datePath>. Run the digest skill first.`

### 2. Parse digest KO section

- Locate the block between `<!-- LANG:KO:START -->` and `<!-- LANG:KO:END -->`.
- Extract the categories line (`> N건 정리 | ...`) for the cover fallback.
- Extract each article's title (from `### {title}` in `## 📋 간단 요약`) and its bullet list.

### 3. Generate Korean headers (per-bullet first, then main cover title)

Follow `references/prompt-headers.md`.

- **Step A — per-bullet headers**: for each bullet of each article, produce one short Korean header that summarises **that specific bullet** and fits a single line on the slide. **Hard cap: 13 Korean characters per header.** Each bullet of the same article must get a clearly different header.
- **Step B — main cover title**: only after the bullet headers are settled, synthesize the day's most important pattern into **one concise Korean main title**. This is not a list of article summaries. It should extract the core signal across the day's items and make a reader want to open the deck.
- Write to `meta/card-news/headers/<datePath>.json`:

  ```json
  {
    "articles": [
      ["기사 1 - 불렛 1 헤더", "기사 1 - 불렛 2 헤더", "..."],
      ["기사 2 - 불렛 1 헤더", "..."]
    ],
    "cover": ["오늘의 메인 제목"]
  }
  ```

- `articles.length` must equal the digest article count exactly.
- `articles[i].length` must equal article *i*'s bullet count exactly.
- `cover.length` must equal **1**.

### 4. Generate English image queries

Follow `references/prompt-image-queries.md`.

- Produce one cover query from the deck's categories.
- Produce one query per article in the same order as the KO summary section.
- Write to `meta/card-news/queries/<datePath>.json`:

  ```json
  { "cover": "...", "articles": ["...", "..."] }
  ```

- If `--reuse-queries` is set and the file exists, skip this step.

### 5. Run the renderer

- From the daily-insights repo root, execute the repository wrapper only:
  ```bash
  scripts/automation/render-cardnews.sh <datePath>
  ```
- Do **not** call `node`, `npm`, `npx`, or `tsx` directly for rendering. The wrapper owns Node PATH bootstrapping, `.env` loading, image API environment checks, renderer execution, and `credits.json` validation.
- The renderer reads both sidecars (`headers/`, `queries/`), resolves images via Unsplash → Pexels → Wikimedia (CC0 / CC BY only), caches downloads under `assets/cache/`, and writes one cover + **one content slide per summary bullet** + `credits.json`. Slides for the same article share the same header and background image.
- If `UNSPLASH_ACCESS_KEY` / `PEXELS_API_KEY` env vars are unset, the pipeline still runs (Wikimedia-only or built-in gradient fallback) — never fail for missing keys.

### 6. Slide layout

**Cover**: renders the one main title from `headers.cover[0]`, then auto-wraps and auto-shrinks the font so it fits on the cover (e.g. `AI가 일하는 방식을 다시 쓴다`).

**Content** (one template): every content slide is one bullet of one article.

- **Header**: bullet-specific short Korean line from `headers.articles[i][j]` (single line, ≤ 13자), placed in the lower-middle of the canvas (~60% height).
- **Body**: that one digest summary bullet, rendered as text directly under the header.

Per-article fan-out: an article with N summary bullets produces N content slides, in the digest's bullet order, **each with its own different header**, all sharing the same background image. Summary and insight are merged into this single template — there is no separate insight slide.

### 7. Completion report

Report:

- Slide count and output directory.
- List of slide numbers where `status: "missing"` in credits (image resolution failed).
