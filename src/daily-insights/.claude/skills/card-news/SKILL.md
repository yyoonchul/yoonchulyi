# Card News Skill

Convert a day's bilingual digest (`content/YYYY/MM/DD.md`) into article-level Instagram card news decks (1080x1350 JPEGs) with contextual backgrounds.

Each article deck is **1 cover + one content slide per top-level detailed note item**. So an article with 5 numbered items under `## 📝 상세 정리` produces 6 slides: `article-cover + 5 content pages`. The cover title and content-page headers are short Korean lines pre-generated by this skill before the renderer runs. Detailed content text is not rewritten by the LLM; it is carried over directly from the digest's indented detail bullets.

## Inputs

- `content/YYYY/MM/DD.md` — existing bilingual digest produced by the `digest` skill.
- Optional arg: `datePath` in `YYYY/MM/DD`. Defaults to today (KST).

## Outputs

- `card-news/article-headers/YYYY/MM/DD.json` — Korean cover titles, original-title/source lines, and content-page headers for article-level decks.
- `card-news/queries/YYYY/MM/DD.json` — English image search queries (daily cover fallback + per-article cover + per-content-page queries).
- `card-news/output/YYYY/MM/DD/A-B.jpg` — article-level deck slides, where `A` is the article index and `B` is the card number inside that article deck (`1-1.jpg`, `1-2.jpg`, `3-4.jpg`, etc.).
- `card-news/output/YYYY/MM/DD/article-NN-credits.json` — per-slide resolved image metadata for article *NN*.
- `public/daily-insights/YYYY/MM/DD/cardnews/A-B` and `A-B.jpg` — GitHub Pages public URL assets for Instagram automation.

## Workflow

1. **Resolve target date.** Use `datePath` arg or today in KST. Confirm `content/<datePath>.md` exists; otherwise abort with clear message.
2. **Parse the KO section** of the digest. Confirm today's digest exists, then extract each article's title, category, summary bullets, and `## 📝 상세 정리` block. Within each article's detailed block, parse each top-level numbered item as one content page; its indented bullets become the body text for that page.
3. **Generate Korean article headers.** Follow `prompt-article-headers.md`.
   - Use the LLM only for the article cover title (`articles[i].cover`), the orange original-title/source line (`articles[i].source`), and each content-page header (`articles[i].sections[j]`), shortened from that numbered detail item's main title.
   - `articles[i].source` must keep the original digest title exactly inside straight double quotes and add a short modifier naming the source, author, site, format, or public figure where useful.
   - When a header/source line names or refers to a person, use the person's name, public handle, role title, or organization label instead of gendered pronouns (`그`, `그녀`, `he`, `she`, `him`, `her`).
   - Do not use the LLM to rewrite detailed body content. The renderer uses the detail bullets from the digest as-is.
   - Save to `card-news/article-headers/<datePath>.json`.
   - `articles.length` must equal the article count exactly. Each `sections.length` must equal that article's top-level detailed item count exactly.
4. **Generate English image queries.** Follow `prompt-image-queries.md`. Produce one daily cover fallback query, plus one cover query and one query per content page for each article. One content page should search and render one image.
5. **Run the article renderer.** From the daily-insights repo root, execute the repository wrapper only:
   ```bash
   scripts/automation/render-cardnews-article.sh <datePath> [article-index]
   ```
   Do **not** call `node`, `npm`, `npx`, or `tsx` directly for rendering. The renderer resolves images via Wikimedia (CC0/CC BY only) → Openverse → NASA Images → Unsplash → Pexels. It also writes public copies to `public/daily-insights/<datePath>/cardnews/` so GitHub Pages can serve `/daily-insights/<datePath>/cardnews/A-B`.
6. **Report.** Output JPEG count, output directory, and any slides with `status: "missing"` in credits.

## Article Deck Layout

**Article cover**: renders the generated article title from `article-headers.articles[i].cover`. The orange mono line directly under the main header uses `article-headers.articles[i].source`; it must include the original digest title exactly inside straight double quotes and usually name the source, author, site, format, or public figure (for example `TechCrunch 기사 "..."`, `GitHub 레포 "..."`, `논문 "..."`, `Will Hackett의 "..."`). The bottom subtitle is `이윤철이 오늘 읽은 아티클`.

**Article content**: parses `## 📝 상세 정리`. Each top-level numbered item becomes one content slide. `article-headers.articles[i].sections[j]` becomes the slide header, shortened from the numbered item title. The indented bullet details are joined underneath without LLM rewriting. Each content slide resolves its own image from `queries.articles[i].sections[j]`.

## Environment

- Built in, no key: Wikimedia Commons, Openverse, NASA Images.
- Optional: `UNSPLASH_ACCESS_KEY`, `PEXELS_API_KEY`. Without them, resolver still tries the no-key sources, then the built-in gradient. Never fail the pipeline on missing keys.
- Editorial/news image APIs such as Guardian Open Platform, NewsAPI, Reuters/AP/Getty-style providers, Bing Image Search, or Google Programmable Search should be treated as opt-in only unless the usage rights are explicitly cleared for publishing. They can surface fresher article-context images, but the returned image URL usually does not grant reuse rights by itself.
