# Card News Skill

Convert a day's bilingual digest (`content/YYYY/MM/DD.md`) into an Instagram card news deck (1080×1350 PNGs) with contextual backgrounds.

Each deck is **1 cover + one content slide per bullet**. So an article with 3 summary bullets produces 3 content slides (each with its own Korean header, all sharing the same background image). The cover title and per-bullet headers are short Korean lines pre-generated by this skill before the renderer runs.

## Inputs

- `content/YYYY/MM/DD.md` — existing bilingual digest produced by the `digest` skill.
- Optional arg: `datePath` in `YYYY/MM/DD`. Defaults to today (KST).

## Outputs

- `meta/card-news/headers/YYYY/MM/DD.json` — Korean short headers (per-bullet + one main cover title).
- `meta/card-news/queries/YYYY/MM/DD.json` — English image search queries (cover + per-article).
- `meta/card-news/output/YYYY/MM/DD/NN.png` — slides in upload order.
- `meta/card-news/output/YYYY/MM/DD/credits.json` — per-slide resolved image metadata.

## Workflow

1. **Resolve target date.** Use `datePath` arg or today in KST. Confirm `content/<datePath>.md` exists; otherwise abort with clear message.
2. **Parse the KO section** of the digest. Extract cover categories and, per article, the title + category + bullets.
3. **Generate Korean headers (per-bullet first, then main cover title).** Follow `prompt-headers.md`.
   - **Step A — per-bullet headers**: for each bullet of each article, write a short Korean header that summarises **that specific bullet** (≤ **13 characters**, single line). All bullet headers within the same article must be visibly different — they should distinguish each angle, not paraphrase the article title.
   - **Step B — main cover title**: only after all bullet headers are settled, synthesize the day's most important pattern into **one concise Korean main title**. This is not a list of article summaries. It should extract the core signal across the day's items and make a reader want to open the deck.
   - Save to `meta/card-news/headers/<datePath>.json` with shape:
     ```json
     {
       "articles": [
         ["기사 1 - 불렛 1 헤더", "기사 1 - 불렛 2 헤더", "..."],
         ["기사 2 - 불렛 1 헤더", "..."]
       ],
       "cover": ["오늘의 메인 제목"]
     }
     ```
   - `articles[i].length` must equal article *i*'s bullet count exactly. `cover.length` must equal **1**.
4. **Generate English image queries.** Follow `prompt-image-queries.md`. Produce one cover query (from categories) and one query per article. Save to `meta/card-news/queries/<datePath>.json`:
   ```json
   { "cover": "...", "articles": ["...", "..."] }
   ```
   Skip regeneration if file already exists and user passed `--reuse-queries`.
5. **Run the renderer.** From the daily-insights repo root, execute the repository wrapper only:
   ```bash
   scripts/automation/render-cardnews.sh <datePath>
   ```
   Do **not** call `node`, `npm`, `npx`, or `tsx` directly for rendering. The wrapper owns Node PATH bootstrapping, `.env` loading, image API environment checks, renderer execution, and `credits.json` validation. The renderer reads both sidecars (`headers/`, `queries/`), resolves images via Unsplash → Pexels → Wikimedia (CC0/CC BY only), and writes one cover + **one content slide per bullet** (article bullets ÷ 1 each) + `credits.json`. Slides for the same article share the same header and background image.
6. **Report.** Output PNG count, output directory, and any slides with `status: "missing"` in credits.

## Slide layout

**Cover**: renders the one main title from `headers.cover[0]`, then auto-wraps and auto-shrinks the font so it fits on the cover (e.g. `AI가 일하는 방식을 다시 쓴다`).

**Content**: every content slide is one bullet of one article and uses a single template:

- **Header**: bullet-specific short Korean line from `headers.articles[i][j]` (one line, ≤13자), placed in the lower-middle of the canvas (~60% height).
- **Body**: that one digest summary bullet, rendered as text directly under the header.

Per-article fan-out: an article with N summary bullets produces N content slides, in the digest's bullet order, **each with its own different header**, all sharing the same background image. There is no separate insight slide — summary and insight are collapsed into this single template.

## Environment

- Optional: `UNSPLASH_ACCESS_KEY`, `PEXELS_API_KEY`. Without them, resolver falls back to Wikimedia, then built-in gradient. Never fail the pipeline on missing keys.
