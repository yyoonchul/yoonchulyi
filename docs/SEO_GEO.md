# SEO and GEO Operating Guide

This guide records the search and AI-discovery rules for `yoonchulyi.com`.
Use it whenever the site structure changes, a new page type is added, or a new
blog post is published.

## Goals

- Make `yoonchulyi.com` the canonical source for Yoonchul Yi, 이윤철, MaekNote,
  and related writing.
- Improve discoverability for English and Korean search queries.
- Keep blog posts and Daily Insights readable by search engines and AI
  retrieval systems without relying on hidden keyword tricks.
- Preserve the current content unless a task explicitly asks to rewrite it.

## Positioning

The site should consistently describe Yoonchul Yi as:

> Yoonchul Yi is an AI product builder working on local-first productivity,
> AI-native notes, Claude Code workflows, and startup/product research.

Primary topic pillars:

- AI-native productivity
- Local-first notes
- Claude Code workflows
- AI agents
- DevTools
- Startups and product research
- Robotics and vertical software

## Language Strategy

The current implementation uses a v1 bilingual strategy:

- One URL may contain English and Korean sections.
- Page-level `lang` should use the primary/default visible language.
- English and Korean body sections should remain identifiable in markup where
  practical.
- Open Graph should include the primary locale and an alternate locale.

Future v2 strategy, if Korean/Naver search becomes a larger priority:

- Split pages into `/en/...` and `/ko/...` routes.
- Add `hreflang` links between language variants.
- Keep one canonical URL per language version.

Do not mix language-specific SEO titles casually. If frontmatter is extended in
the future, prefer explicit fields such as `seoTitleEn`, `seoTitleKo`,
`descriptionEn`, and `descriptionKo`.

## Blog Post Requirements

Blog posts should work even with only the legacy fields:

```yaml
title: "Post title"
date: 2026-05-18
```

For new posts, prefer the expanded SEO frontmatter:

```yaml
title: "Human-readable title"
seoTitle: "Search-friendly title with the main topic"
description: "A concise 140-160 character description of the post."
slug: "stable-lowercase-slug"
datePublished: "2026-05-18"
dateModified: "2026-05-18"
language: "en"
tags:
  - Claude Code
  - Local Markdown
topics:
  - claude-code
  - local-first-productivity
featured: false
index: true
```

Rules:

- `seoTitle` should be clear to a searcher, not only poetic or personal.
- `description` should summarize the specific argument or experience in the post.
- `slug` should be stable, lowercase, and hyphenated for all new posts.
- `tags` are display/search concepts; `topics` are future internal topic hubs.
- Set `index: false` only for drafts, thin pages, or private/utility posts.
- Do not change existing post body content just to satisfy SEO. Improve metadata
  first.

## Structured Data

The SEO backbone should continue to emit:

- `WebSite` on the home page.
- `Person` with `@id: https://yoonchulyi.com/#person`.
- `ProfilePage` on About.
- `BlogPosting` for blog posts and Daily Insight detail pages.
- `BreadcrumbList` for navigable pages.
- `ItemList` for blog and Daily Insight indexes.

All article authors should reference the same person entity:

```json
{ "@id": "https://yoonchulyi.com/#person" }
```

This keeps future posts connected to the same author/entity graph.

## Sitemap and Robots

The site should expose:

- `/sitemap.xml`
- `/robots.txt`
- `/llms.txt`

Sitemap rules:

- Include home, about, blog index, Daily Insights index, all indexable blog
  posts, and Daily Insight detail pages.
- Use `dateModified` for blog `lastmod` when present.
- Fall back to `datePublished` or `date`.
- Exclude blog posts where `index: false`.

Robots rules:

- Allow general search crawlers.
- Allow AI search/retrieval crawlers such as `OAI-SearchBot`,
  `ChatGPT-User`, `Claude-SearchBot`, `Claude-User`, and `PerplexityBot`.
- Training crawlers such as `GPTBot` and `ClaudeBot` may be blocked if the site
  owner wants search visibility without model-training opt-in.

## Daily Insights

Daily Insights can be indexed, but they should not look like thin autogenerated
archives over time.

Current backbone rules:

- Detail pages should emit `BlogPosting` schema.
- Index pages should emit `ItemList` schema.
- Titles and descriptions should be generated from the English heading/content
  when frontmatter is unavailable.

Future content rules:

- Add a clear title that includes the date and main topics.
- Keep original source links visible.
- Add original perspective when possible.
- If a page is only a thin link summary, consider `index: false` or consolidation
  into a larger digest.

## Site Expansion Rules

When adding new page types:

- Use `BaseLayout` and pass explicit `title`, `description`, and
  `canonicalPath`.
- Add JSON-LD through the shared schema helpers in `src/lib/schema.ts`.
- Add pages to `/sitemap.xml` generation when they should be indexed.
- Use stable lowercase URLs for new evergreen pages.
- Prefer internal links between blog posts, project pages, topic pages, and Daily
  Insights.

Recommended future routes:

- `/projects/`
- `/projects/maeknote/`
- `/topics/`
- `/topics/claude-code/`
- `/topics/local-first-productivity/`
- `/topics/ai-agents/`

Topic pages should connect long-form essays with related Daily Insights and
projects.

## Verification Checklist

Before shipping SEO/GEO changes:

- Run `npm run build`.
- Confirm `/sitemap.xml` contains indexable blog posts.
- Confirm `/robots.txt` is generated in the built site.
- Inspect at least one blog post and verify:
  - canonical URL
  - meta description
  - `og:type="article"`
  - `BlogPosting` JSON-LD
  - no accidental `noindex`
- After deployment, check the live URLs:
  - `https://yoonchulyi.com/robots.txt`
  - `https://yoonchulyi.com/sitemap.xml`
  - one representative blog post
- Submit or resubmit the sitemap in Google Search Console and Naver Search
  Advisor when available.
